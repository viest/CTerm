# CTerm

CTerm 是一个面向 AI 编程工作流的原生 macOS 终端应用。它把多项目管理、Git worktree、Agent 预设、分屏终端和项目级自动脚本放进同一个 AppKit 窗口里，目标不是做一个通用终端，而是做一个适合持续编码、并行开工和上下文切换的工作台。

## 这个项目在解决什么问题

常见的 AI 编程流程往往分散在几个地方：

- 终端里跑 agent
- Finder 或编辑器里切项目
- 手动创建和清理 worktree
- 自己记每个 agent 的启动命令
- 每次切换任务都重新搭一遍环境

CTerm 试图把这些动作压缩成一个连续的桌面体验：选项目、开 workspace、启动 agent、并行分屏、查看状态、执行项目脚本，都在同一个窗口里完成。

## 项目特点

### 1. 面向 AI 编程的终端工作台

顶部预设栏可以直接启动常用 agent，而不是每次手敲一长串命令。预设支持名称、命令、描述、提供商和是否显示在主窗口顶部，适合把 Claude、Codex、Gemini、Aider 这类工具固定成团队或个人工作流的一部分。

### 2. 基于 Git worktree 的 workspace 工作流

新建 workspace 时，应用会直接为项目创建独立 worktree，并把 agent、分支和工作目录绑在一起。这个设计天然适合并行处理多个任务，避免在同一份工作树里频繁切分支、清理脏状态。

### 3. 多标签页与多分屏终端

CTerm 支持终端标签页、水平/垂直分屏、焦点切换和 pane 尺寸调整。它不是把多个 shell 简单摆在一起，而是允许你把一个任务拆成多个协作视角，例如：

- 一个 pane 跑 agent
- 一个 pane 看测试或日志
- 一个 pane 手工执行修复命令

### 4. Ghostty 驱动的终端渲染

终端视图不是手写字符栅格，而是直接集成 Ghostty 渲染层。这样可以获得更现代的终端行为、更稳定的输入体验和更接近真实终端的交互语义，同时仍然保留原生 AppKit 外壳和窗口管理能力。

### 5. 项目级自动脚本

项目目录下可以放 `.cterm/config.json`，为 workspace 定义 `setup`、`teardown` 和 `run` 脚本。CTerm 会在合适的生命周期自动执行这些命令，并注入：

- `CTERM_ROOT_PATH`
- `CTERM_WORKSPACE_NAME`
- `CTERM_WORKSPACE_PATH`

这让项目初始化、依赖预热、开发服务启动或清理动作可以内建到工作流里，而不是靠口头约定。

### 6. 项目级与全局 agent 预设并存

除了全局预设，项目还可以提供自己的 `.cterm/presets.json`。这意味着每个仓库都可以定义最适合自己的 agent 启动方式，而全局预设继续保留个人常用工具，两者会在主窗口里合并展示。

### 7. 面向真实开发场景的侧边栏

主窗口不仅有终端，还包含项目列表、Workspace 管理、文件浏览、变更视图和端口面板。目标是减少“切出终端再去别处确认信息”的次数，让常见上下文都留在当前窗口内。

### 8. 状态、用量和性能监控

应用内置 token usage、provider 状态和性能快照的展示能力。对于长期运行的 agent 会话，这些信息不是装饰，而是决定是否继续跑、是否切模型、是否需要回收上下文的重要依据。

### 9. 设置项围绕长期使用优化

设置窗口支持：

- Agent 预设编辑
- 顶部预设栏显示控制
- 终端字体、字号、主题、光标样式
- 通用路径与行为设置
- 快捷键查看与重置

这些能力都服务于一个目标：让 CTerm 适合高频、长时间使用，而不是一次性演示。

## 典型使用方式

一个比较自然的使用路径是：

1. 添加本地项目
2. 为任务创建独立 workspace
3. 选择一个 agent 预设直接启动
4. 需要时拆出多个 pane 同时观察输出、编辑和验证
5. 通过项目脚本自动完成 setup / run / teardown
6. 在同一个窗口里查看变更、文件和运行状态

它适合以下场景：

- 同时推进多个 issue 或多个 agent 任务
- 需要频繁创建临时分支和 worktree 的仓库
- 想把 AI 编程命令固定成稳定预设，而不是靠复制粘贴
- 希望终端、项目导航和工作区状态放在一个原生窗口里

## 当前定位

CTerm 不是一个“什么都做”的通用终端模拟器。它更接近一个围绕 AI 编程工作流定制的 macOS 开发台：

- 终端是核心
- project/workspace 是一级概念
- agent 不是插件，而是主流程的一部分
- Git worktree 不是附加功能，而是基础工作单元

如果你的日常工作已经明显依赖 agent、并行任务和项目级自动脚本，这个项目的设计方向会比传统终端更贴近实际使用。
